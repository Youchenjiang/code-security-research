# 0.1 威脅建模與攻擊面分析 (Threat Modeling & Attack Surface Analysis)

## 📌 核心精神 (Core Concept)
在程式碼撰寫前的架構設計階段，系統性地分析潛在攻擊路徑、元件信任邊界與資料流，以識別安全威脅並指導安全測試策略。

## 🔍 子技術流派 (Sub-Genres)
### Threat Modeling Frameworks (威脅建模框架)
- **簡介**: 結構化的設計期威脅識別。代表框架：STRIDE、PASTA、LINDDUN（隱私威脅）。

### Attack Tree Analysis (攻擊樹分析)
- **簡介**: 以樹狀圖建模攻擊者達成目標的各種潛在路徑，估算攻擊機率與成本。


## 🏷️ 學術常用關鍵字 (Keywords)
`threat modeling`, `STRIDE`, `trust boundary`, `attack tree`, `PASTA framework`, `data flow diagram security`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`Microsoft Threat Modeling Tool`, `OWASP Threat Dragon`, `pyTM`, `IriusRisk`
