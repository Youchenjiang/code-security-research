# 2.1 Web與API動態漏洞掃描 (DAST)

## 📌 核心精神 (Core Concept)
在不獲取原始碼或程式內部運行狀態的情況下，模擬外部攻擊者對運行中的 Web 應用或 API 發送惡意請求，分析回應特徵（如狀態碼、內容、延遲）以識別漏洞（如 SQL 注入、XSS、IDOR、邏輯缺陷）。

## 🔍 子技術流派 (Sub-Genres)
### Web 應用與 API 差異測試
- **簡介**: 對 API 注入相同請求，比較授權前後或多個端點的回應差異，用於發現 IDOR 或邏輯漏洞。代表工具：RESTler、EvoMaster、PrediQL。

### 傳統 DAST 黑箱掃描
- **簡介**: 模擬外部攻擊者對執行中的應用發動攻擊測試。代表工具：OWASP ZAP、Burp Suite、Contrast DAST。


## 🏷️ 學術常用關鍵字 (Keywords)
`REST API testing`, `GraphQL fuzzing`, `differential response`, `semantic-aware API testing`, `DAST`, `web vulnerability scanning`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`OWASP ZAP`, `Burp Suite`, `RESTler`, `EvoMaster`
