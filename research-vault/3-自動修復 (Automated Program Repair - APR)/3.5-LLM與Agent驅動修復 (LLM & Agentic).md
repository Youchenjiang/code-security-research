# 3.5 大語言模型與 Agent 驅動修復 (LLM-based & Agentic APR)

## 📌 核心精神 (Core Concept)
LLM 在無需微調的情況下，通過 Zero/Few-shot Prompting 直接生成修復補丁；進一步整合工具呼叫、測試執行反饋、多輪迭代，形成自主 Agent 閉環，是目前前沿趨勢。

## 🔍 子技術流派 (Sub-Genres)
### LLM Zero/Few-shot Prompting 修復
- **簡介**: 直接以漏洞報告、錯誤訊息、程式碼上下文組成 Prompt 餵入 LLM。代表工具：ChatRepair

### LLM 迭代反饋修復 (Iterative Feedback Loop)
- **簡介**: LLM 生成補丁 → 執行測試 → 將失敗資訊回饋給 LLM → 重新生成。代表工具：LLM4CVE、RAVEN。

### 多 Agent 自動化修復系統 (Multi-Agent Agentic APR)
- **簡介**: 建立多個專責 Agent（故障定位、補丁生成、測試驗證）協同解決複雜跨文件問題。代表工具：SWE-agent、AutoCodeRover、AutoPatch、AgenticVM、RAVEN。

### RAG 輔助修復 (Retrieval-Augmented Repair)
- **簡介**: 以 RAG 從歷史 CVE 補丁庫、CWE 知識庫中檢索相關修復案例。代表工具：RAVEN、VulKey。
