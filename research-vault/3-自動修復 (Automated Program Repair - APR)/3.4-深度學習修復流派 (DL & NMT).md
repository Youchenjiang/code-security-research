# 3.4 深度學習修復流派 (DL-based / NMT-based APR)

## 📌 核心精神 (Core Concept)
將程式碼修復視為神經機器翻譯（NMT）問題——把「有 Bug 的程式碼」翻譯成「修復後的程式碼」，以大量歷史補丁對訓練模型，自動學習修復規律。

## 🔍 子技術流派 (Sub-Genres)
### Seq2Seq / 早期 NMT 修復
- **簡介**: 以 Encoder-Decoder 架構將 buggy 程式碼編碼，解碼生成 fixed 程式碼。代表工具：SequenceR、CoCoNuT、DLFix。

### Graph / AST-aware DL 修復
- **簡介**: 以程式結構圖（AST、CFG）而非純文字序列作為輸入，讓模型學習程式的結構性語意。代表工作：DEAR (Tree-LSTM)、Recoder、CURE。

### 預訓練程式碼語言模型微調 (Fine-tuned PLM for APR)
- **簡介**: 在大規模程式碼語料上預訓練，再微調於特定漏洞修復資料集。代表工作：VulRepair、InferFix、TFix。


## 🏷️ 學術常用關鍵字 (Keywords)
`neural machine translation repair`, `encoder-decoder bug fix`, `sequence-to-sequence APR`, `tree-based repair`, `AST-aware fix`, `multi-hunk repair`, `pre-trained model APR`, `CodeT5 vulnerability repair`, `fine-tuning bug fix`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`SequenceR`, `CoCoNuT`, `DEAR`, `Recoder`, `VulRepair`, `InferFix`
