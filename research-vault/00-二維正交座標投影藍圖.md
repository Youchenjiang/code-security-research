# 🗺️ 程式碼安全研究流派：2D 象限座標投影設計藍圖

本文件定義了知識庫中 39 個二級流派節點在 **2D 直角座標系（象限圖）** 上的投影設計與自動生成規範。

> [!WARNING]
> **定位聲明**：本 2D 象限座標圖僅作為「學術地圖與輔助教學導航」之用，以幫助讀者直觀瞭解流派大體分佈。由於圖形渲染加入了微距避撞擾動（Jitter），若需要進行嚴謹的學術數據檢索與自動化篩選，**請一律以各流派 Markdown 筆記內的 Frontmatter 4D Facet 數據為準**。

## 1. 設計動機與背景 (Design Rationale)

在安全軟體工程中，傳統樹狀目錄無法表現流派的多維度屬性，若直接用網路拓撲圖呈現，又會退化為難以閱讀的「髮線團 (Hairball)」。

為此，我們設計了 **2D 象限直角座標投影**，並制定了嚴格的 **數據驅動生成規則**，以解決三大核心問題：
1. **解決維度擁擠**：棄用無區分度的「測試抽象層級」，改用程式分析最根本的 **「精度與運行開銷 (Precision vs. Scalability)」** 作為 Y 軸，使節點能均勻散布。
2. **避免主觀臆估（假精確性）**：所有節點的「基礎座標（Base Coordinates）」皆由 `setup_vault.py` 中的規則代碼自動運算生成，不含人工隨意給分，保證評分一致性 (Inter-rater reliability)。
3. **解決文字重疊**：在渲染時，由編譯腳本對重合於同一坐標點的節點自動套用「圓周微距避撞演算法（Jitter）」，在視覺上推開標籤。

## 2. 座標代碼運算規則 (Programmatic Derivation Rules)

我們在 `setup_vault.py` 中實現了以下兩套量化映射公式，將 4D Facet 資料轉換為離散的序位刻度 (Ordinal Scale)。

### X 軸公式：分析手段 (Analysis Methodology)
基礎座標僅分為三個離散值：`0.2` (靜態)、`0.5` (混合)、`0.8` (動態)。
* **規則**：
  1. 檢索 methodology Facet，若包含 `"定位與診斷分析 (FL)"` 或 `"修復與補丁合成 (APR)"` $\rightarrow$ 定位為 **混合 (Hybrid) `X = 0.5`**。
  2. 若 methodology 僅包含靜態分析相關屬性 $\rightarrow$ 定位為 **靜態 (Static) `X = 0.2`**。
  3. 若 methodology 僅包含動態/模糊測試/插樁監控相關屬性 $\rightarrow$ 定位為 **動態 (Dynamic) `X = 0.8`**。

### Y 軸公式：分析精度與運行開銷 (Precision & Execution Cost)
基礎座標僅分為三個離散值：`0.2` (輕量/低開銷)、`0.5` (中等複雜度)、`0.8` (重型/高精確)。
* **規則**：根據技術範式的運算資源與保真度進行映射：
  * **重型與精準 (`Y = 0.8`)**：包含定理證明、狀態空間窮舉、符號執行約束求解、指令級污點傳播 (DTA)、執行期編譯插樁監控 (ASan)、主動防禦加固 (CFI)、自動漏洞利用 (AEG)。
  * **中等複雜度 (`Y = 0.5`)**：包含全局圖查詢、型別校驗、跨程序資料流、API規則審計、執行期行為沙盒、單元級動態測試生成 (ATG)、統計故障定位、LLM多輪反饋 Agent 修復。
  * **輕量與高速 (`Y = 0.2`)**：包含語法 AST 匹配、SCA依賴庫查表、歷史commit挖掘、大模型單輪代碼生成、高速隨機/反饋模糊測試、Web黑箱流量掃描。

## 3. 避碰撞渲染演算法 (Visual Jittering Algorithm)

由於 39 個節點在基礎座標上會收攏至 `3x3` 共 9 個離散網格點上，直接渲染會導致標籤完全重合。

我們在 `setup_vault.py` 中實現了以下避碰撞算法：
1. **分組**：將所有基礎座標 $(BaseX, BaseY)$ 相同的節點歸入同一群組。
2. **圓周分佈**：若群組內節點數 $N > 1$，則將它們以 $Base$ 座標為圓心，`0.04` 為半徑，平均分布於圓周上。
3. **計算公式**：對群組中第 $i$ 個節點（$0 \le i < N$），其渲染座標 $(RenderX, RenderY)$ 計算如下：
   $$\theta_i = i \times \frac{2\pi}{N}$$
   $$RenderX = BaseX + 0.04 \times \cos(\theta_i)$$
   $$RenderY = BaseY + 0.04 \times \sin(\theta_i)$$
4. **邊界裁剪**：將結果裁剪至 $[0.01, 0.99]$ 區間，防止標籤超出圖表邊緣。

## 4. 座標演化與治理規則 (Governance Rules)

當未來有新論文引入，或需要增加新的流派節點時，必須遵守以下治理規則：

1. **依賴關係不變性**：
   * 嚴禁手動微調 `00-多維正交二維座標圖.md` 中的座標。所有變更必須透過修改 `setup_vault.py` 中 `get_base_coordinates` 函數的規則判定來實現。
2. **分類變更回填**：
   * 若流派的 methodology Facet 屬性發生變更，必須同步重新運行 `setup_vault.py`。腳本會依據 X 軸判定公式自動重算落點，確保 X/Y 座標與 Frontmatter 數據始終維持 100% 的追溯一致性。
