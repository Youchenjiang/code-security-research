# 1.2 資料流分析 (Data Flow Analysis)

## 📌 核心精神 (Core Concept)
追蹤資料在程式中如何流動，特別關注「外部輸入」是否未經驗證便流入危險操作（如資料庫查詢、系統命令）。這是靜態分析中最核心的技術族群。

## 🔍 子技術流派 (Sub-Genres)
### 污點分析 (Taint Analysis)
- **簡介**: 將外部輸入標記為「污染源（Source）」，追蹤其是否抵達「危險接收點（Sink）」而未經清理（Sanitizer）。代表工具：CodeQL、Pixy、FlowDroid、IRIS。

### 別名分析 / 指標分析 (Alias / Pointer Analysis)
- **簡介**: 分析「哪些指標指向同一塊記憶體」，是精準 taint 分析的前提。代表工具：LLVM Alias Analysis、IDEal。

### 跨程序分析 (Interprocedural Analysis)
- **簡介**: 跨越函式邊界追蹤資料流，處理 call graph 的建構與摘要（Summary）。


## 🏷️ 學術常用關鍵字 (Keywords)
`taint analysis`, `information flow`, `source-sink`, `IFDS/IDE algorithm`, `sanitization`, `points-to analysis`, `alias-aware dataflow`, `context-sensitive analysis`, `flow-sensitive analysis`, `call graph construction`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`CodeQL`, `Pixy`, `FlowDroid`, `LLVM Alias Analysis`, `IDEal`
