# 1.12 靜態污點分析 (Static Taint Analysis)

## 📌 核心精神 (Core Concept)
專門追蹤外部輸入（Source）在不執行程式下，如何沿變數與函式呼叫路徑傳播至敏感接收點（Sink），著重於污點傳播規則與隱含流處理。

## 🔍 子技術流派 (Sub-Genres)
### Source-to-Sink Taint Tracking (源頭到接收點追蹤)
- **簡介**: 在控制流與資料流圖上尋找未經過濾的污點傳播路徑。代表工具：FlowDroid、CodeQL、Joern Taint Engine。

### Implicit Flow Analysis (隱含流分析)
- **簡介**: 檢測並分析透過條件分支與控制流隱含洩漏的資料流。


## 🏷️ 學術常用關鍵字 (Keywords)
`static taint analysis`, `source-sink tracking`, `explicit taint flow`, `implicit information flow`, `sanitization verification`, `CodeQL taint queries`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`CodeQL`, `FlowDroid`, `Joern`, `Fortify`
