# 1.10 密碼學與協議安全審計 (Cryptographic & Protocol Security)

## 📌 核心精神 (Core Concept)
專注於檢測密碼學 API 誤用（如弱金鑰、靜態 IV、不安全填充）以及協議實現中控制流與狀態機的正確性與時序安全。

## 🔍 子技術流派 (Sub-Genres)
### 密碼學 API 誤用檢測 (Cryptographic API Misuse)
- **簡介**: 靜態分析程式碼中密碼學函數的調用參數，核對安全規則。代表工具：CogniCrypt、CrySL、CryptoLint。

### 時序與恆定時間驗證 (Constant-time Verification)
- **簡介**: 分析代碼執行時間是否與輸入數據相關，防止時序側通道攻擊。代表工具：dudect、CacheD。


## 🏷️ 學術常用關鍵字 (Keywords)
`cryptographic API misuse`, `weak key detection`, `constant-time implementation`, `timing leak verification`, `protocol state machine`, `SSL/TLS configuration audit`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`CogniCrypt`, `CrySL`, `CryptoLint`, `dudect`, `CacheD`
