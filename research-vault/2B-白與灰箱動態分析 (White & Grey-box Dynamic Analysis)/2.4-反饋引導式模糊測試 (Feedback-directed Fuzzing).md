# 2.4 反饋引導式模糊測試 (Feedback-directed Fuzzing)

## 📌 核心精神 (Core Concept)
透過編譯期插樁或硬體追蹤，動態獲取程式執行時的內部資訊（如分支覆蓋率），以此引導模糊器朝向未探索的路徑進行演化變異，是目前發現未知漏洞最有效的方法。

## 🔍 子技術流派 (Sub-Genres)
### 覆蓋率引導灰箱模糊測試 (CGF)
- **簡介**: 動態收集邊緣覆蓋率（Bitmap）作為演化反饋。代表工具：AFL、AFL++、libFuzzer、honggfuzz。

### 定向灰箱模糊測試 (DGF)
- **簡介**: 引導測試輸入往特定代碼位置（如 patch 點、特定 API）集中探索。代表工具：AFLGo、Hawkeye、BEACON。

### 核心模糊測試 (Kernel Fuzzing)
- **簡介**: 以系統呼叫序列為輸入，動態測試 OS 核心並解決崩潰隔離與硬體追蹤。代表工具：syzkaller、kAFL。

### 結構與文法感知模糊測試 (Structure/Format-aware)
- **簡介**: 理解複雜輸入格式（PDF/XML/AST），進行有語義的變異。代表工具：Nautilus、WEIZZ。

### LLM/ML 輔助模糊測試
- **簡介**: 利用模型生成種子、引導變異或理解 API 規格。代表工具：LLaMAFuzz、MTFuzz。
