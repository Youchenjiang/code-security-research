# 2.8 語意差異與並發偵測 (Differential & Concurrency)

## 📌 核心精神 (Core Concept)
透過對多個等價實作進行差異比對以發現語意缺陷，或動態監控多執行緒交錯，捕捉並發相關的非確定性漏洞。

## 🔍 子技術流派 (Sub-Genres)
### 編譯器與虛擬機差異測試
- **簡介**: 自動生成程式並比較不同編譯器優化等級或 VM 實作的輸出差異。代表工具：Csmith、Rustlantis、EVMFuzz。

### 並發與競爭條件偵測
- **簡介**: 追蹤記憶體存取與鎖定，偵測 data race, deadlock 等並發缺陷。代表工具：TSan、Helgrind、CHESS、RAZZER。

### 記憶體安全動態精密檢測
- **簡介**: 動態精密校驗 heap/stack 的安全邊界（如 Valgrind/Memcheck, Dr. Memory）。


## 🏷️ 學術常用關鍵字 (Keywords)
`differential testing`, `compiler testing`, `random program generation`, `EVM differential testing`, `data race detection`, `happens-before`, `lockset algorithm`, `shadow memory`, `memory error detection`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`Csmith`, `Rustlantis`, `EVMFuzz`, `TSan`, `Valgrind/Memcheck`, `Dr. Memory`
