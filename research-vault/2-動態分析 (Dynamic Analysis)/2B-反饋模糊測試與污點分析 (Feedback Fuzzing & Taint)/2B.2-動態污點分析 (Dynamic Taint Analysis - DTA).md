# 2B.2 動態污點分析 (Dynamic Taint Analysis - DTA)

## 📌 核心精神 (Core Concept)
在程式執行時，對每個暫存器或記憶體位元組維護污染標籤（Taint Tag），精確追蹤污染源資料流向危險接收點（Sink）的實際執行路徑。精準度高但具有顯著的運行期開銷。

## 🔍 子技術流派 (Sub-Genres)
### 全系統動態污點追蹤 (Whole-System DIFT)
- **簡介**: 在仿真器（如 QEMU）層插樁，追蹤跨程式、跨系統呼叫的資料流。代表工具：DECAF、DECAF++、TEMU。

### 使用者空間動態污點分析
- **簡介**: 利用動態二進位插樁（DBI）在使用者空間追蹤單一程序的資料流.代表工具：libdft、Minemu。

### 資訊流安全與側通道檢測
- **簡介**: 追蹤機密資料是否在執行時洩漏至時間、分支等可觀測側通道。


## 🏷️ 學術常用關鍵字 (Keywords)
