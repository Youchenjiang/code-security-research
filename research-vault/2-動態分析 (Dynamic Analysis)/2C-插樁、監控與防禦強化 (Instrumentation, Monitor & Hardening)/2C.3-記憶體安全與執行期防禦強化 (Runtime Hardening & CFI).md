# 2C.3 記憶體安全與執行期防禦強化 (Runtime Hardening & CFI)

## 📌 核心精神 (Core Concept)
評估並測試部署在實際生產環境中的執行期硬體/軟體安全加固技術（如控制流完整性 CFI、堆疊金絲雀、隨機化佈局）的實用效能與繞過方法。

## 🔍 子技術流派 (Sub-Genres)
### Control-Flow Integrity (控制流完整性)
- **簡介**: 在編譯或執行期驗證間接跳轉目標位址是否在合法 CFG 內。代表技術：LLVM CFI。

### Memory Safety Hardening (記憶體加固)
- **簡介**: 使用 hardened allocator 或硬體能力（如 CHERI）防止過隨與釋放後使用漏洞。代表工具：hardened_malloc、mimalloc。


## 🏷️ 學術常用關鍵字 (Keywords)
`control-flow integrity`, `CFI`, `ASLR bypass`, `runtime defense`, `hardened malloc`, `shadow stack`, `CHERI capability`, `memory safety protection`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`LLVM CFI`, `hardened_malloc`, `CHERI SDK`, `mimalloc`
