# 2C.1 執行期插樁與監控 (Instrumentation & Sanitizers)

## 📌 核心精神 (Core Concept)
在編譯期、載入期或執行期將檢測邏輯注入目標程式中，以在執行時監控程式行為、捕捉違規行為或主動進行防禦。

## 🔍 子技術流派 (Sub-Genres)
### 編譯期 Sanitizers
- **簡介**: 編譯時插入輕量檢查指令，高效偵測記憶體與未定義行為。代表工具：ASan (Address)、MSan (Memory)、TSan (Thread)、UBSan。

### 動態二進位插樁 (DBI)
- **簡介**: 在二進位運行時動態插入分析代碼（JIT重編），免代碼。代表框架：Intel PIN、DynamoRIO、Valgrind、Frida。

### 互動式應用安全測試 (IAST)
- **簡介**: 應用 runtime 內置 agent，結合真實流量在程式內部精確定位漏洞行號。代表工具：Contrast Security、Seeker。

### 執行期自我防禦 (RASP)
- **簡介**: 部署於生產環境的 agent，實時監控並攔截 SQL 注入或 RCE 等惡意操作。代表工具：OpenRASP、Imperva。


## 🏷️ 學術常用關鍵字 (Keywords)
`shadow memory`, `compile-time instrumentation`, `address sanitizer`, `dynamic binary instrumentation`, `IAST`, `runtime agent`, `runtime protection`, `self-protection`

## 🛠️ 代表性工具 / 框架 (Representative Tools)
`ASan`, `TSan`, `Intel PIN`, `DynamoRIO`, `Frida`, `Contrast Security`, `OpenRASP`
